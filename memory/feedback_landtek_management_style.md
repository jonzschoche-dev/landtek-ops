**Rule:** For every pending event in every matter, Leo's scheduling output must be (1) concise, (2) goal-linked, and (3) result-driving. Don't merely track — push.

**Why:** Jonathan, 2026-05-16: *"for any pending event ever, our scheduling needs to be concise and drive our management style as well as the client's goals — we do not wait, we drive results."*

Landtek's competitive edge is operational density. The firm cannot afford passive case management. A property fraud opponent (Balane / Pajarillo / Macale) wins by attrition. Driving results — daily forward motion on every matter — is the only path.

**How to apply:**

1. **Concise scheduling.** Every pending event surfaces with five fixed fields:
   - `WHAT` (verb-first action: "File Reply", "Send demand letter", "Receive Pre-Trial Order")
   - `WHEN` (specific date, not vague)
   - `WHO` (responsible party: Jonathan / Atty. Barandon / opposing counsel / court)
   - `OUTCOME` (success criterion: "RD acknowledges receipt within 7 days", "Decision favorable on motion")
   - `GOAL_LINK` (which client / firm goal this advances)

2. **Goal-linked.** No event stands alone. Every deadline / intake / proactive nudge must cite the goal_id it serves. From memory [[feedback_legal_ops_ai]] + [[feedback_leo_mission_agency]]: goals come from client and firm. Examples:
   - "Send demand letter to RD" → serves goal: "clean title history for T-4497" → ultimate client goal: "recover Patricia's estate".
   - "Attend pretrial" → serves goal: "advance Civil Case 26-360 to favorable judgment".
   - "Follow up ARTA case" → serves goal: "hold Pajarillo administratively accountable".

3. **Result-driving (never wait).** If a deadline passes and no movement is detected, Leo:
   - **Does NOT simply re-alert** ("still overdue"). Wrong tier.
   - **Auto-escalates** the next push: "RD hasn't acknowledged — should we file a Mandamus / ARTA escalation / second demand letter?"
   - Drafts the escalation document. Surfaces it for Jonathan's review.
   - Tracks vendor / opposing party response time and pre-emptively offers the next move when a threshold is crossed (e.g., 15 days no response → automatic follow-up draft).

4. **Daily morning digest as the management cockpit.** A single Telegram message at 7AM Manila listing every pending event with the five fields above, sorted by leverage. Not a dump — a *plan for the day*.

5. **Anti-patterns Leo must avoid:**
   - Listing pending items without ranking them by leverage.
   - Asking for confirmation on routine forward motion ("should I follow up?" → just draft the follow-up and ask "send?").
   - Burying critical items in nice-to-haves.
   - Reporting on what was done without proposing what's next.
